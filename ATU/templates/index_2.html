Сайыс туралы

«ATU & KazNU AI» цифрлық шешімдер сайысының мақсаты – тағам өндірісі, қаржы саласы, мейрамхана бизнесі, туризм және қызмет көрсету индустриясындағы өзекті мәселелерді шешуге бағытталған заманауи цифрлық және жасанды интеллектке негізделген IT-шешімдерді әзірлеу, білім алушылардың инновациялық ойлау қабілетін дамыту және олардың тәжірибелік құзыреттіліктерін жетілдіру.

Қатысушылар сайыс барысында

  • Нақты бизнес-кейстер мен салалық мәселелерге талдау жүргізеді
  • Қысқа мерзім ішінде жасанды интеллект элементтерін қолдана отырып цифрлық өнімнің прототипін әзірлейді
  • Жобаларын сараптамалық комиссия алдында таныстырып, қорғайды

Негізгі міндеттері

  • Экономика салаларында цифрлық трансформация үдерістерін қолдау
  • Жасанды интеллект технологияларын тәжірибеде қолдануды ынталандыру
  • Стартап мәдениетін қалыптастыру және кәсіпкерлік бастамаларды дамыту
  • Командалық жұмыс, жобалық менеджмент және креативті ойлау дағдыларын жетілдіру
  • IT саласында қолданбалы, бәсекеге қабілетті шешімдер әзірлеуге жағдай жасау

Задания

Задание 1: Умный контроль качества Python · Jupyter

Суть задачи

Разработать инструмент для базового анализа производственных показателей с целью контроля качества продукции.

Задача системы

  • ▸Найти отклонения в производственных параметрах
  • ▸Посчитать процент бракованной продукции
  • ▸Построить наглядные графики для анализа

Фактически решается задача

Построения первичного дашборда и расчетной модели для оценки качества.

Входные данные

Таблица: температура, влажность, время, статус

Требования к решению

Инструменты: Python, Jupyter Notebook.

Формат результата

Файл .py или .ipynb с дашбордом

Критерии оценки

✅ Расчёты — 40% 📊 Визуализация — 30% 💡 Выводы — 30%

Ожидаемый результат

Готовый Python-файл(.py, .ipynb), способный на основе введенных параметров автоматически рассчитывать метрики брака и визуализировать отклонения.

Задание 2: Анализ производственной линии Python · pandas

Суть задачи

Автоматизировать процесс выявления производственных аномалий на основе сырых данных с датчиков, используя скрипты.

Задача системы

  • ▸Обработать предоставленный массив данных
  • ▸Выявить аномалии по заданным правилам и алгоритмам

Фактически решается задача

Построения pipeline для фильтрации и программного анализа показаний датчиков.

Входные данные

CSV-файл с данными производственных датчиков

Требования к решению

Инструменты: Python (с использованием библиотеки pandas).

Формат результата

Скрипт .py или .ipynb с pipeline

Критерии оценки

🗂 Обработка данных — 40% 🔍 Логика аномалий — 30% 📋 Вывод — 30%

Ожидаемый результат

Воспроизводимый Python-скрипт, который очищает данные, применяет правила поиска аномалий и формирует отчет по отклонениям.

Задание 3: ML-модель брака Python · sklearn

Суть задачи

Разработать и внедрить алгоритм машинного обучения для проактивного выявления дефектной продукции.

Задача системы

  • ▸Обучить модель ML для предсказания появления брака
  • ▸Провести анализ точности предсказаний

Фактически решается задача

Создания предиктивной ML-модели для классификации статуса продукции.

Входные данные

Исторические данные производственного процесса

Требования к решению

Инструменты: Python + библиотека sklearn.

Формат результата

Код модели + веса .pkl + метрики

Критерии оценки

🏗 Архитектура — 40% 🎯 Точность — 30% 🔬 Анализ — 30%

Ожидаемый результат

Обученная ML-модель, способная предсказывать брак на новых данных с подтвержденной метриками точностью.

Задание 4: IoT-система ESP32 · MicroPython

Суть задачи

Спроектировать аппаратно-программный комплекс для мониторинга состояния физического объекта в реальном времени.

Задача системы

  • ▸Осуществлять мониторинг показателей в реальном времени
  • ▸Генерировать сигнал тревоги при критических значениях

Фактически решается задача

Построения связки "железо - софт" для реализации концепции Интернета вещей (IoT).

Входные данные

Потоковые данные с микроконтроллера или эмулятора

Требования к решению

Инструменты: ESP32 (или эмуляция) + язык Python.

Формат результата

Код C++/MicroPython + Python-сервер + демо

Критерии оценки

🏗 Архитектура — 40% ⚙️ Реализация — 40% 🎬 Демо — 20%

Ожидаемый результат

Работающий прототип IoT-системы (физический или эмулированный), успешно считывающий данные и триггерящий тревогу по заданному условию.

Задание 5: Цифровой двойник Python · Grafana

Суть задачи

Создать виртуальную копию физической установки, синхронизированную с её процессами для визуализации и анализа.

Задача системы

  • ▸Симулировать работу физической установки
  • ▸Обеспечить наглядную визуализацию процессов в Grafana

Фактически решается задача

Разработки математической/логической симуляции и интеграции её с профессиональными инструментами мониторинга.

Входные данные

Параметры физической модели (задаются программно)

Требования к решению

Инструменты: Python (для логики) / Grafana (для визуализации).

Формат результата

Дашборд Grafana + Python-скрипт источника данных

Критерии оценки

🧮 Симуляция — 40% 📈 Grafana — 30% 🔁 Логика — 30%

Ожидаемый результат

Полноценный интерактивный дашборд цифрового двойника, отображающий состояние симулируемой установки в реальном времени.

Задание 6: Предиктивное управление ML · Reinforcement Learning

Суть задачи

Разработать автономную интеллектуальную систему, способную не только анализировать, но и принимать решения по управлению сложным процессом.

Задача системы

  • ▸Предсказывать дальнейшее поведение процесса
  • ▸Автоматически управлять процессом на основе предсказаний

Фактически решается задача

Внедрения передовых алгоритмов искусственного интеллекта для создания замкнутого цикла управления без участия человека.

Входные данные

Динамические данные среды/процесса в реальном времени

Требования к решению

Инструменты: Machine Learning (ML) / Reinforcement Learning (RL).

Формат результата

AI-агент + документация + результаты тестирования

Критерии оценки

💡 Концепт — 40% ⚙️ Реализация — 30% 🚀 Новизна — 30%

Ожидаемый результат

Инновационная система предиктивного управления, демонстрирующая способность агента самостоятельно оптимизировать или корректировать заданный процесс с использованием методов RL/ML.
